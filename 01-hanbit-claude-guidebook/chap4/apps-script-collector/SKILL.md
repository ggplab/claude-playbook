# Apps Script 데이터 수집기 만들기

## 이 스킬이 하는 일

구글 시트와 앱스 스크립트만으로, 사람이 손대지 않아도 매일 정해진 시각에 데이터를 시트에 쌓는
자동 수집 시스템을 설계·구현해 준다. 대상은 주식 시세가 대표적이지만, `GOOGLEFINANCE`로
가져올 수 있는 것(주가·지수·환율 등)이면 무엇이든 같은 틀로 만들 수 있다.

대상 사용자는 대체로 **코드를 모른다.** 그래서 결과물은 "복사해서 붙여넣기만 하면 되는" 형태여야
하고, 설명은 전부 일상 언어로 써야 한다. 전문 용어는 최소화하고, 쓰더라도 짧게 풀어 준다.

## 산출물 3종 (항상 이 순서)

1. **PRD 문서** (`chapN_PRD.md` 또는 사용자가 지정한 이름) — 무엇을 만들지 고정하는 기획서.
2. **붙여넣기용 앱스 스크립트 코드** (`.gs`) — 코드 한 번 붙여넣기로 탭·종목·자동 실행이 스스로 생김.
3. **설치·실행 가이드** (`.md`) — 사용자가 직접 할 일만 순서대로 정리한 짧은 안내.

각 단계는 앞 단계를 그대로 반영해야 한다. 특히 **PRD의 예외 처리 규칙은 코드에 100% 반영**되어야
하고, 코드에 넣은 메뉴/동작은 설치 가이드에 그대로 설명되어야 한다. 세 문서가 서로 어긋나면 안 된다.

## 1단계: 입력 받기 (무엇을·어디서·언제)

작업 전, 아래 세 가지를 확정한다. 사용자가 이미 말했으면 다시 묻지 말고 그대로 쓴다. 빠진 것만 묻되,
사용자가 자리를 비운 정황(스케줄 실행 등)이면 합리적 기본값을 정하고 그 가정을 문서 맨 앞에 밝힌다.

- **무엇을(수집 항목)**: 종목/대상과 가져올 항목. 주식 기본값은 종가·전일대비·등락률·시가·고가·저가·거래량·날짜.
- **어디서(데이터 출처)**: 기본은 `GOOGLEFINANCE("KRX:종목코드", 항목)`. 해외/환율이면 코드 형식만 다름.
- **언제(실행 시각)**: 기본은 장 마감 후. 한국 증시는 15:30 마감이라 **매일 오후 4시**가 안전한 기본값.

## 2단계: PRD 문서 작성

`references/prd_structure.md`를 읽고 그 구성을 따른다. PRD는 반드시 다음 6개 절로 구성한다:
목표 / 시트 구성(탭·컬럼) / 수집 규칙 / 자동 실행 방식 / 예외 처리 / 내가 직접 해야 하는 일.

**예외 처리 절에는 7개 규칙을 빠짐없이 담는다** (원문과 근거는 `references/prd_structure.md` 참고):
값 지연 시 재시도, 전일대비는 등락률에서 역산, 일부 항목이 비어도 줄 전체 실패 금지, 주말·공휴일·중복
날짜 건너뛰기, 트리거 중복 생성 방지, 시트·탭·헤더 자가 생성, 모든 오류는 '로그' 탭 기록.

코드를 모르는 사람이 읽을 문서이므로 일상 언어로 쓰고, 컬럼 구성이나 항목 매핑은 표로 정리한다.
`references/prd_example.md`가 삼성전자·SK하이닉스 기준 완성 예시이니 형식 참고용으로 읽는다.

## 3단계: 앱스 스크립트 코드 생성

`references/apps_script_template.gs`가 완성 템플릿이다. 이걸 기반으로, 1단계에서 받은 수집 대상·
출처·시각에 맞게 `SYMBOLS`, 헤더, `TRIGGER_HOUR` 등만 바꿔서 낸다. 템플릿은 다음을 이미 만족한다:

- **붙여넣기 한 번으로 완성**: 시트를 새로고침하면 상단에 `주식 수집기` 메뉴가 생기고, `① 최초 설정`
  하나로 탭·헤더·자동 실행이 스스로 만들어진다.
- **메뉴 4종**: ① 최초 설정 · ② 지금 수집 · ③ 자동 실행 켜기/끄기 · ④ 문제 진단.
- **예외 규칙 7개가 모두 코드에 반영**됨 (각 함수에 어떤 규칙이 들어갔는지는 템플릿 주석 참고).

`GOOGLEFINANCE`는 셀 함수라 앱스 스크립트에서 직접 못 부른다. 그래서 숨김 임시 탭에 공식을 써서
계산시키고, 값이 뜰 때까지 재시도해 읽는 방식(`fetchQuote_`)을 쓴다 — 이 패턴을 유지한다.

## 4단계: 설치·실행 가이드 작성

`references/install_guide_template.md`를 따른다. **사용자가 직접 할 일만** 순서대로, 짧게 쓴다.
빠뜨리면 안 되는 항목: 시트 시간대를 (GMT+09:00) 서울로 맞추기(오후 4시가 한국 시간이 되도록),
코드 전체 붙여넣기, 새로고침 후 메뉴 확인, `① 최초 설정`에서 권한 승인, `② 지금 수집`으로 검증.
메뉴 4종의 용도와 "값이 이상하면 로그 탭을 보라"는 안내도 포함한다.
