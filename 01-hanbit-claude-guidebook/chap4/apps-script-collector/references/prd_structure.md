# PRD 구성 및 예외 처리 규칙 (참조)

이 스킬로 만드는 PRD 문서는 아래 6개 절 구성을 그대로 따른다. 코드를 모르는 사람이 읽는
문서이므로 전부 일상 언어로 쓰고, 컬럼·항목 매핑은 표로 정리한다.

## PRD 6개 절 구성

1. **목표** — 사람이 손대지 않아도 정해진 시각에 데이터가 시트에 한 줄씩 쌓이게 하는 것. 성공 기준을
   한 문장으로 명시(예: "평일 오후 4시마다 각 종목 시세가 정확히 한 줄씩 쌓인다").
2. **시트 구성(탭·컬럼)** — 종목별 데이터 탭(헤더 8개 컬럼), 로그 탭(시각·종목·구분·내용),
   계산용 임시 탭. 각 컬럼의 이름과 의미를 표로.
3. **수집 규칙** — 데이터 출처(`GOOGLEFINANCE("KRX:코드", 항목)`), 가져올 항목과 GOOGLEFINANCE
   이름 매핑(종가=price, 등락률=changepct, 시가=priceopen, 고가=high, 저가=low, 거래량=volume),
   날짜 형식(YYYY-MM-DD), 저장 방식(하루 한 종목당 한 줄, 함수 결과가 아닌 고정 값으로 저장).
4. **자동 실행 방식** — 시간 기반 트리거로 매일 지정 시각(기본 오후 4시) 자동 실행. 트리거는 스크립트가
   스스로 등록. 중복 방지 규칙은 예외 처리 절 참조.
5. **예외 처리** — 아래 7개 규칙(원문 유지).
6. **내가 직접 해야 하는 일** — 시트 만들기 → 시간대 설정 → 코드 붙여넣기 → 권한 승인 → 자동 실행
   켜기 → 다음 날 확인. 사람이 하는 일만 순서대로.

## 예외 처리 7개 규칙 (반드시 코드에 100% 반영)

1. **값이 늦게 뜨면 잠시 기다렸다가 재시도할 것.** GOOGLEFINANCE 값은 실행 직후 아직 계산 중
   (`Loading...`)일 수 있다. 곧바로 실패로 보지 말고 잠깐 기다린 뒤 다시 읽는다. 여러 번 재시도해도
   비어 있으면 그때 로그에 남긴다. → 코드: `fetchQuote_`의 재시도 루프(`MAX_TRIES`, `WAIT_MS`).

2. **전일대비는 등락률에서 역산할 것.** 국내 주식은 전일 종가가 직접 안 들어온다. 종가와 등락률로 계산:
   전일 종가 = 종가 ÷ (1 + 등락률/100), 전일대비 = 종가 − 전일 종가. → 코드: `collectAll_`의 change 계산.

3. **일부 항목이 비어도 그 줄 전체를 실패 처리하지 말 것.** 한 항목(예: 거래량)만 안 들어와도 나머지는
   정상 저장하고, 빈 항목만 로그에 남긴다. → 코드: 항목별 `isNum_` 검사 후 빈칸 처리 + missing 로그.

4. **주말·공휴일과 이미 저장된 날짜(중복)는 건너뛸 것.** 장이 안 열리는 날은 건너뛰고, 오늘 날짜가 이미
   저장돼 있으면 다시 추가하지 않는다. → 코드: `isWeekend_`, `isHoliday_`, `hasDate_`.

5. **자동 실행은 여러 번 설정해도 중복 생성되지 않게 할 것.** 트리거를 만들기 전에 같은 트리거가 이미
   있는지 확인하고, 있으면 새로 만들지 않는다. → 코드: `hasTrigger_` 확인 후 `ensureTrigger_`.

6. **시트·탭·헤더가 없으면 스스로 만들 것.** 종목 탭·로그 탭·헤더가 없으면 스크립트가 자동 생성한다.
   → 코드: `ensureAllSheets_`, `getOrCreateDataSheet_`, `ensureHeader_`.

7. **모든 오류는 별도 '로그' 탭에 남길 것.** 값 실패·계산 실패·중간 오류는 물론, 정상 실행·건너뜀도
   기록해 시트만 봐도 무슨 일이 있었는지 알 수 있게 한다. → 코드: `log_(종목, 구분, 내용)`.
