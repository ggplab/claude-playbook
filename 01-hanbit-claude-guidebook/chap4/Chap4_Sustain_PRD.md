# 한국 주식 시세 수집기 — PRD (제품 요구사항 정의서)

문서명: chap4_PRD.md
작성일: 2026-07-10 (개정: 주간 이메일 리포트 추가)
스택: 구글 시트(Google Sheets) + 앱스 스크립트(Apps Script)
대상 종목: 삼성전자, SK하이닉스
자동 실행:
- 시세 수집 — 매일 오후 4시(장 마감 후)
- 주간 리포트 — 매주 월요일 오전 8시, 지난주 요약을 내 Gmail로 발송

---

## 1. 목표

매일 장이 끝난 뒤, 삼성전자와 SK하이닉스의 시세를 사람이 손대지 않아도 자동으로 구글 시트에 쌓이게 하고, 매주 월요일 아침에는 지난주 요약을 이메일로 받아보는 것이 목표다.

- 구글 시트 안에서 `GOOGLEFINANCE` 함수로 시세를 직접 가져온다. 별도의 외부 프로그램이나 유료 데이터 서비스는 쓰지 않는다.
- 매일 오후 4시에 앱스 스크립트가 스스로 실행되어, 그날 시세 한 줄을 데이터 탭에 추가한다.
- 매주 월요일 오전 8시에 앱스 스크립트가 지난주(월~일) 시세를 요약해 내 Gmail로 보낸다.
- 하루가 지나면 어제 줄은 그대로 남고 오늘 줄이 새로 붙는 식으로, 날짜별 기록이 계속 누적된다.
- 코드를 모르는 사람도 시트만 열면 종목별 시세 흐름을 표로 확인할 수 있어야 한다.

성공 기준: 사람이 아무것도 누르지 않아도, 평일 오후 4시마다 각 종목의 그날 시세가 정확히 한 줄씩 시트에 쌓이고, 월요일 오전 8시마다 지난주 요약 메일이 도착한다.

---

## 2. 시트 구성 (탭·컬럼)

구글 시트 하나 안에 탭(하단 시트 이름표)을 세 종류로 나눈다.

### 2-1. 데이터 탭 — `삼성전자`, `SK하이닉스`

종목마다 탭을 하나씩 둔다. 각 탭은 아래 8개 컬럼(열)을 가진다. 맨 윗줄(1행)은 헤더(제목 줄)이고, 그 아래로 날짜별 기록이 한 줄씩 쌓인다.

| 컬럼 | 헤더 이름 | 설명 |
|------|-----------|------|
| A | 날짜 | 시세 기준일 (예: 2026-07-10) |
| B | 종가 | 그날 마지막 거래 가격 |
| C | 전일대비 | 전날 종가 대비 오른/내린 금액 |
| D | 등락률 | 전날 대비 변동 비율(%) |
| E | 시가 | 그날 장 시작 가격 |
| F | 고가 | 그날 최고 가격 |
| G | 저가 | 그날 최저 가격 |
| H | 거래량 | 그날 거래된 주식 수 |

### 2-2. 로그 탭 — `로그`

자동 실행이 언제 돌았는지, 문제가 있었는지를 남기는 탭이다. 사람이 원인을 확인할 때 이 탭만 보면 된다. 주간 리포트 발송 결과(성공/실패)도 이 탭에 남는다.

| 컬럼 | 헤더 이름 | 설명 |
|------|-----------|------|
| A | 시각 | 기록이 남은 날짜·시간 |
| B | 종목 | 어떤 종목 처리 중이었는지 (리포트는 '리포트') |
| C | 구분 | 정보 / 경고 / 오류 중 하나 |
| D | 내용 | 무슨 일이 있었는지 사람이 읽을 수 있는 설명 |

### 2-3. 종목 목록 (선택)

나중에 종목을 늘리기 쉽도록, 종목 이름과 KRX 코드를 한곳에 적어두는 방식을 권장한다. 이번 단계에서는 삼성전자(`KRX:005930`), SK하이닉스(`KRX:000660`) 두 종목만 대상으로 한다.

주간 리포트는 별도 탭을 만들지 않고, 위 데이터 탭의 기록을 읽어 요약해 메일로만 보낸다.

## 3. 수집 규칙

무엇을, 어디서, 어떻게 가져오는지에 대한 규칙이다.

- **데이터 출처**: 모든 시세는 구글 시트의 `GOOGLEFINANCE("KRX:종목코드", 항목)` 함수로 가져온다.
  - 삼성전자 = `KRX:005930`
  - SK하이닉스 = `KRX:000660`
- **가져오는 항목**과 GOOGLEFINANCE에서 쓰는 이름:
  - 종가 = `price` (실시간/최근 체결가)
  - 등락률 = `changepct` (전날 대비 변동 %)
  - 시가 = `priceopen`
  - 고가 = `high`
  - 저가 = `low`
  - 거래량 = `volume`
- **전일대비**는 GOOGLEFINANCE가 국내 주식에서 직접 주지 않는다. 그래서 등락률과 종가를 이용해 스크립트가 직접 계산한다. (자세한 계산 규칙은 5절 예외 처리 참고.)
- **날짜**는 실행된 날의 날짜를 기준으로 하며, 형식은 `YYYY-MM-DD`(예: 2026-07-10)로 통일한다.
- **저장 방식**: 하루에 한 종목당 한 줄. 값을 가져온 뒤에는 함수 결과가 아니라 그때의 숫자 값(고정값)으로 저장한다. 그래야 다음 날 시세가 바뀌어도 어제 기록이 어제 값 그대로 남는다.
- **저장 위치**: 각 종목 데이터 탭의 맨 아래 빈 줄에 새 줄로 추가한다.

---

## 4. 자동 실행 방식

두 가지 자동 실행을 둔다. 둘 다 앱스 스크립트의 **시간 기반 트리거(time-driven trigger)**를 쓰며, 스크립트가 스스로 등록한다. 사람이 매번 버튼을 누를 필요가 없다.

- **① 매일 시세 수집** — **매일 오후 4시경**에 수집 함수가 실행된다. (한국 증시는 오후 3시 30분에 마감되므로, 마감 후 값이 안정된 시점이다.) 실행되면 대상 종목을 하나씩 돌며 그날 줄을 추가하고 로그를 남긴다.
- **② 주간 리포트 발송** — **매주 월요일 오전 8시**에 리포트 함수가 실행된다. 지난주(직전 월요일~일요일) 데이터 탭 기록을 종목별로 요약해, 내 Gmail(시트 소유자 계정)로 보낸다.

두 트리거의 중복 방지 규칙은 5절 예외 처리에 명시한다.

### 4-1. 주간 리포트에 담기는 내용

각 종목에 대해 지난주 기록에서 다음을 계산해 이메일 표로 보낸다: 시작 종가(그 주 첫 기록), 종료 종가(그 주 마지막 기록), 주간 등락(금액·%), 주간 최고가·최저가, 평균 거래량, 수집된 날 수.

설치 초기처럼 **지난주 기록이 하나도 없으면**, 가장 최근에 수집된 날이 속한 주(월~일)로 자동 대체해 그 주 데이터를 보낸다. 이때 메일 상단에 "지난주 데이터가 없어 최근 주간으로 대체했다"는 안내를 표시한다. 아직 아무 데이터도 수집되지 않았을 때만 "데이터 없음"으로 표기한다.

## 5. 예외 처리

아래 규칙은 스크립트가 반드시 지켜야 하는 동작이다.

1. **값이 늦게 뜨면 잠시 기다렸다가 재시도할 것.** GOOGLEFINANCE 값은 실행 직후 아직 계산 중(`Loading...`)일 수 있다. 이 경우 곧바로 실패로 보지 말고, 잠깐 기다린 뒤 다시 값을 읽는다. 몇 번 재시도해도 계속 비어 있으면 그때 로그에 남긴다.

2. **전일대비는 등락률에서 역산할 것.** 국내 주식은 전일 종가가 직접 들어오지 않으므로, 전일대비를 직접 가져올 수 없다. 대신 종가와 등락률로 계산한다.
   - 전일 종가 = 종가 ÷ (1 + 등락률/100)
   - 전일대비 = 종가 − 전일 종가
   - 즉, 등락률과 종가만 정상이면 전일대비는 스크립트가 스스로 만들어 채운다.

3. **일부 항목이 비어도 그 줄 전체를 실패 처리하지 말 것.** 예를 들어 거래량 하나만 안 들어와도, 나머지 값(종가·시가 등)은 정상적으로 저장한다. 비어 있는 칸은 빈 칸으로 두고, 어떤 항목이 비었는지만 로그에 남긴다. 한 항목 때문에 그날 기록 전체를 버리지 않는다.

4. **주말·공휴일과 이미 저장된 날짜(중복)는 건너뛸 것.** 토요일·일요일 등 장이 열리지 않는 날은 수집을 건너뛴다. 또한 오늘 날짜가 이미 그 종목 탭에 저장돼 있으면, 다시 추가하지 않고 건너뛴다. (같은 날짜 줄이 두 번 생기지 않도록.) 건너뛴 경우에도 로그에 "건너뜀" 사유를 남긴다.

5. **자동 실행은 여러 번 설정해도 중복 생성되지 않게 할 것.** 트리거를 만들기 전에 이미 같은 트리거가 등록돼 있는지 먼저 확인한다. 이미 있으면 새로 만들지 않는다. 이 규칙은 매일 수집 트리거와 주간 리포트 트리거 모두에 똑같이 적용한다. 그래야 설정을 여러 번 실행해도 수집은 하루 한 번, 리포트는 주 한 번만 돈다.

6. **시트·탭·헤더가 없으면 스스로 만들 것.** 실행 시 필요한 종목 탭(삼성전자·SK하이닉스)이나 로그 탭이 없으면 스크립트가 자동으로 만든다. 헤더(제목 줄)가 비어 있으면 2절에 정의한 컬럼 이름을 채워 넣는다. 사람이 미리 표를 그려두지 않아도 동작해야 한다.

7. **모든 오류는 별도 '로그' 탭에 남길 것.** 값이 끝내 안 들어왔거나, 계산이 안 됐거나, 중간에 문제가 생긴 경우 등 모든 상황을 로그 탭에 기록한다. 정상 실행도 "몇 시에 어떤 종목을 저장했다"는 식으로 남겨, 나중에 시트만 봐도 무슨 일이 있었는지 알 수 있게 한다.

### 5-1. 주간 리포트 관련 예외 (위 규칙의 연장)

- **지난주 데이터가 하나도 없어도 실패로 보지 말 것.** 먼저 가장 최근 수집된 주간으로 대체해 발송을 시도하고, 그것마저 없으면 "데이터 없음"으로 정상 발송한다. 어느 경우든 그 사실을 로그에 남긴다. (규칙 3·7의 연장)
- **메일 발송이 실패해도 시스템 전체를 멈추지 말 것.** 발송 오류는 로그 탭에 '오류'로 남기고, 다음 실행에 영향이 없게 한다. (규칙 7)
- **요약 계산 시 비어 있는 값은 건너뛸 것.** 특정 날의 고가·거래량 등이 비어 있어도 그 값만 빼고 최고가·평균 등을 계산한다. (규칙 3)

## 6. 내가 직접 해야 하는 일

코드는 스크립트가 처리하지만, 사람이 최초 한 번 해줘야 하는 준비 작업이 있다. 순서대로 하면 된다.

1. **구글 시트 새로 만들기.** 구글 드라이브에서 빈 스프레드시트를 하나 만들고, 알아보기 쉬운 이름(예: "한국주식 시세수집")으로 저장한다. 탭이나 표는 미리 만들 필요 없다(스크립트가 만들어 준다).

2. **표준 시간대 확인.** 시트 `파일 → 설정`에서 시간대를 (GMT+09:00) 서울로 맞춘다. 그래야 "오후 4시", "월요일 오전 8시"가 한국 시간이 된다.

3. **앱스 스크립트 열기.** 그 시트에서 상단 메뉴 `확장 프로그램 → Apps Script`를 눌러 스크립트 편집기를 연다.

4. **코드 붙여넣기.** 받은 `code.gs` 내용을 편집기에 전체 복사해 붙여넣고 저장한다.

5. **권한 승인(최초 1회).** 스크립트를 처음 실행하면 구글이 접근 허용을 물어본다. 이번에는 **시트 접근**뿐 아니라 **Gmail로 메일 보내기 권한**도 함께 요청한다(주간 리포트 발송용). 본인 계정으로 승인해 준다. 이 승인이 있어야 자동 실행과 메일 발송이 돌 수 있다.

6. **자동 실행 등록 실행.** `① 최초 설정`을 한 번 실행하면 매일 수집 + 매주 리포트 트리거가 함께 등록된다.

7. **확인.** `② 지금 수집`으로 시세가 쌓이는지, `⑤ 리포트 지금 발송`으로 메일이 오는지 각각 한 번씩 눌러 확인한다. 값이 이상하면 로그 탭과 `④ 문제 진단`을 본다.

정리하면, 사람이 하는 일은 "시트 만들기 → 시간대 설정 → 코드 붙여넣기 → 권한 승인 → 자동 실행 한 번 켜기 → 확인"까지이고, 그 뒤로는 시트가 알아서 쌓이고 월요일마다 메일이 온다.
