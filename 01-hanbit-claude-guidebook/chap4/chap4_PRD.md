# PRD — 한국 주식 시세 수집기 (무료·자동)

작성일: 2026-07-09 · 스택: **Google Sheets + Apps Script + 매일 오후 4시 자동실행** · 데이터 출처: **GOOGLEFINANCE("KRX:종목코드")**

> 이 문서는 "무엇을 만들지"만 정의합니다. 실제 코드는 다음 단계에서 작성합니다.

---

## 1. 개요

Google Sheets와 Apps Script만으로, 별도 비용·서버·유료 API 없이 국내 주식의 일별 시세를 자동으로 수집·누적하는 도구다. 사용자는 "설정" 탭에 관심종목(기본: 삼성전자·SK하이닉스)을 등록해 두기만 하면, 매일 장 마감 후(기본 오후 4시, 한국시간) **날짜·종가·전일대비·등락률·시가·고가·저가·거래량**이 종목별 탭에 한 줄씩 자동으로 쌓인다. 코드를 모르는 사용자를 대상으로 하므로, **코드 붙여넣기 1회와 권한 승인만으로** 탭·종목·자동실행이 스스로 구성되어야 한다.

## 2. 목표와 비목표

**목표**는 무료로 매일 자동 수집되는 국내 주식 시세 이력을 만드는 것이다. 사용자가 종목을 자유롭게 추가·삭제할 수 있어야 하고, 값이 늦게 뜨거나 일부 항목이 비는 국내주식 특성에도 안정적으로 동작해야 하며, 오류가 나도 무엇이 문제인지 사용자가 바로 파악할 수 있어야 한다.

**비목표**는 실시간(초·분 단위) 시세, 해외주식·지수·환율의 기본 지원, 분봉/틱 데이터, 매매 신호·투자 자문이다. 이들은 본 도구의 범위 밖이다.

## 3. 대상 사용자

코드를 모르는 개인 투자자로, 매일 관심종목의 종가와 등락률을 기록으로 남겨 추세를 보고 싶어 하는 사람이다. 스프레드시트 사용에는 익숙하지만 프로그래밍은 하지 않는다.

## 4. 데이터 출처와 항목 정의

시세는 `GOOGLEFINANCE("KRX:종목코드", 항목)` 함수로 가져온다. 무료이고 API 키가 필요 없으며, 약 15~20분 지연된 값을 제공한다. 사용하는 항목은 다음과 같다.

| 시트 컬럼 | GOOGLEFINANCE 항목 | 설명 |
|-----------|--------------------|------|
| 종가 | `price` | 현재가. 장 마감 후 실행하므로 사실상 종가로 간주 |
| 등락률(%) | `changepct` | 전일 종가 대비 등락률(%) |
| 시가 | `priceopen` | 당일 시가 |
| 고가 | `high` | 당일 고가 |
| 저가 | `low` | 당일 저가 |
| 거래량 | `volume` | 당일 거래량 |

### 국내 시장의 중요한 제약

**국내주식은 전일종가(`closeyest`)가 GOOGLEFINANCE로 오지 않는다.** 따라서 **전일대비는 직접 받지 않고 등락률에서 역산한다.** 종가와 등락률(%)만 있으면 다음과 같이 계산된다.

```
전일종가 = 종가 ÷ (1 + 등락률/100)
전일대비 = 종가 × 등락률 ÷ (100 + 등락률)
```

또한 GOOGLEFINANCE는 국내종목에 대해 시가·고가·저가·거래량 등 일부 항목을 제공하지 않을 때가 있다. 이는 오류가 아니라 데이터 원천의 한계이므로, **해당 항목이 비어 있어도 그 행은 정상 저장한다.**

## 5. 시트 구성 (탭·컬럼)

스프레드시트는 다음 탭으로 구성된다.

| 탭 | 생성 방식 | 역할 |
|----|-----------|------|
| **설정** | 최초 설정 시 자동 생성, 이후 사용자가 직접 편집 | 관심종목 목록(종목명·6자리 종목코드)을 담는다 |
| **종목별 데이터** (예: 삼성전자, SK하이닉스) | 자동 생성 | 각 행이 하루치 시세를 담는다 |
| **로그** | 자동 생성 | 실행·재시도·부분누락·오류를 시각과 함께 기록 |
| **_계산** | 자동 생성(숨김) | GOOGLEFINANCE 계산에만 쓰는 임시 탭. 자동 관리 |

### "설정" 탭 컬럼

| 종목명 | 종목코드(6자리) |
|--------|-----------------|
| 삼성전자 | 005930 |
| SK하이닉스 | 000660 |

### 종목별 데이터 탭 컬럼 (열 순서 고정)

| 날짜 | 종가 | 전일대비 | 등락률(%) | 시가 | 고가 | 저가 | 거래량 |
|------|------|----------|-----------|------|------|------|--------|

## 6. 수집 규칙

**6.1 자동 수집** — 매일 지정 시각(기본 오후 4시, 한국시간)에 등록된 모든 관심종목의 시세를 조회해 각 종목 탭에 한 줄씩 추가한다. 장 마감(15:30) 이후 실행이므로 종가가 안정적으로 잡힌다.

**6.2 값 지연 대비 재시도** — GOOGLEFINANCE 값이 즉시 뜨지 않는 경우가 있어, 종가가 비어 있으면 공식을 다시 계산시키며 최대 6회까지(회당 약 3초 대기) 재시도한다. 재시도 사실은 로그에 남긴다.

**6.3 부분 누락 허용** — 핵심 값인 종가조차 못 받은 종목은 그날 그 종목만 건너뛰고(전체 실패로 처리하지 않음) 로그에 남긴다. 시가·고가·저가·거래량 등 일부 항목만 비어 있으면 빈칸으로 두고 행은 정상 저장한다.

**6.4 주말·중복 건너뛰기** — 자동 실행이 토·일요일에 걸리면 수집하지 않고 로그만 남긴다. 이미 오늘 날짜의 행이 존재하는 종목은 중복 저장하지 않고 건너뛴다. (수동 "지금 수집"은 주말에도 실행 가능하다.)

**6.5 공휴일 처리** — 공휴일에는 시세가 갱신되지 않아 전일과 같은 값이 저장될 수 있는데, 이는 정상이며 중복 날짜가 아니면 저장된다(원하면 사용자가 삭제).

**6.6 종목코드 보정** — 종목코드가 6자리 숫자가 아니거나 앞자리 0이 사라진 경우(예: `5930`)는 6자리(`005930`)로 복원한다. 형식이 이상하면 진단에서 경고한다.

## 7. 자동실행 방식

Apps Script의 **시간 기반 트리거(time-driven trigger)** 를 사용한다. "자동실행 켜기"를 누르면 매일 오후 4시(Asia/Seoul)에 수집 함수가 실행되도록 예약이 만들어지고, "자동실행 끄기"를 누르면 그 예약이 삭제된다. 시간대는 Asia/Seoul로 고정한다.

스프레드시트 상단에 **"📈 주가수집기"** 메뉴를 만들고 네 가지 기능을 제공한다.

| 메뉴 | 동작 |
|------|------|
| ① 최초 설정 | 설정·로그 탭, 기본 관심종목(삼성전자·SK하이닉스), 종목별 데이터 탭과 머리글을 자동 구성 |
| ② 지금 수집 | 즉시 한 줄을 쌓는다(동작 확인·수동 수집용) |
| ③ 자동실행 켜기/끄기 | 매일 오후 4시 예약을 만들거나 지운다 |
| ④ 문제 진단 | 탭·종목코드·자동실행 상태와 GOOGLEFINANCE 항목별 실제 수신 여부를 점검해 팝업+로그로 보고 |

모든 주요 동작은 오류가 나면 로그 탭에 시각·구분·내용을 기록한다. (자동 실행 중에는 팝업이 뜨지 않을 수 있어 로그로 남긴다.)

## 8. 비기능 요구사항

무료(Google 계정만 필요)로 동작하고, 코드 붙여넣기 1회와 권한 승인만으로 사용 가능해야 한다. Apps Script의 하루 트리거·실행시간 한도 안에서 동작하며, 종목 수는 수십 개 규모까지 무난하다. 시간대는 Asia/Seoul로 고정한다.

## 9. 내가(사용자가) 직접 해야 하는 일

아래는 **코드를 몰라도 할 수 있는, 사용자가 직접 해야 하는 작업**이다. 나머지(탭 생성·수집·자동실행·오류처리)는 모두 코드가 알아서 한다.

| 단계 | 사용자가 하는 일 | 코드가 필요한가? |
|------|------------------|:----------------:|
| 1 | 구글 드라이브에서 **새 구글 스프레드시트**를 하나 만든다 | ❌ |
| 2 | 상단 메뉴 **확장 프로그램 → Apps Script** 열기 | ❌ |
| 3 | 다음 단계에서 만들 **코드를 붙여넣고 저장**(Ctrl/Cmd+S) | ❌ (붙여넣기만) |
| 4 | 스프레드시트로 돌아가 **새로고침** → 상단 **📈 주가수집기** 메뉴 확인 | ❌ |
| 5 | **① 최초 설정** 클릭 → 최초 1회 **권한 승인**(내 구글 계정으로 접근 허용) | ❌ |
| 6 | **② 지금 수집**으로 값이 잘 쌓이는지 확인 | ❌ |
| 7 | **③ 자동실행 켜기** 클릭 → 이후 매일 오후 4시 자동 수집 | ❌ |
| 8 | (원할 때) **"설정" 탭에 종목명·6자리 코드**를 추가/삭제 | ❌ |

### 권한 승인에 대한 안내

⑤단계에서 구글이 "이 앱이 확인되지 않았습니다" 같은 경고를 보일 수 있는데, 이는 **내가 직접 만든 스크립트라서** 나오는 정상 화면이다. "고급 → (안전하지 않음)으로 이동"을 눌러 **내 스프레드시트와 이메일 접근**을 허용하면 된다. 이 스크립트는 오직 내 시트에만 데이터를 쓰며, 외부로 데이터를 보내지 않는다.

## 10. 사용자 플로우 (요약)

시트 생성 → Apps Script에 코드 붙여넣기·저장 → 시트 새로고침 → **① 최초 설정**(권한 승인) → **② 지금 수집**으로 확인 → **③ 자동실행 켜기**. 이후 매일 오후 4시에 데이터가 자동으로 쌓인다. 종목을 바꾸고 싶으면 "설정" 탭만 편집한다.

## 11. 향후 확장 (선택, 이번 범위 밖)

지수·환율·해외종목 지원, 등락률 급변 시 Gmail 요약 알림, Looker Studio 대시보드 연결, 월간 시트 자동 분리, PER·PBR 등 펀더멘털 추가(OpenDART API) 등을 고려할 수 있으나 본 버전 범위에는 포함하지 않는다.
