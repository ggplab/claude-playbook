## 1. 목표와 비목표

### 우리가 하려는 것 (목표)
- 서울에서 전세(또는 월세)로 살 집을 찾는 **신혼부부**가,
- 지난 1년 실거래 데이터를 보고 **"우리 예산으로 어느 동네·어떤 크기까지 갈 수 있는지"**를 스스로 판단하게 돕는다.
- 값이 **오르는 중인지 내리는 중인지** 흐름을 눈으로 보게 한다.
- 내 컴퓨터에서 **파일 하나를 더블클릭하면 바로 열리는** 웹페이지로 만든다.

### 우리가 하지 않을 것 (비목표)
안 만드는 것을 분명히 해서, 데이터에 없는 걸 억지로 넣지 않습니다.

| 안 하는 것 | 이유 |
|---|---|
| 지도에 매물 찍기 | 데이터에 주소·좌표가 없음 |
| 지하철역·역세권 필터 | 역과의 거리 정보가 없음 |
| 신축/구축 구분 | 건축연도가 없음 |
| 방 개수·구조·향·관리비 표시 | 해당 정보가 없음 |
| **지금 나온 매물 보여주기** | 이 데이터는 "지난 거래 기록"이지 실시간 매물이 아님 |
| 개별 거래 한 건씩 나열 | 요약본만 넣기로 했으므로 개별 건은 저장하지 않음 |
| 인터넷 공개 | 내 컴퓨터에서만 봄 |
| 투자·매수 추천 | 시세를 보여줄 뿐, "여기 사라"는 조언은 하지 않음 |

## 2. 데이터 출처와 스키마

### 출처

- 받은 곳: 깃허브 실습 저장소 `github.com/ggplab/claude-playbook` → `chap5/seoul-apt-latest.csv`
- **주의**: 국토교통부 원본을 가공한 것인지 파일에 표기가 없어 **확인 불가**. 실습용으로는 충분하나, 실제 공개 서비스로 쓸 땐 원본 대조 필요.

### 원본 컬럼 (12개)
| 컬럼 | 뜻 | 비고 |
|---|---|---|
| `contract_ym` | 계약 연월 | 2025-07 ~ 2026-06 |
| `contract_date` | 계약일 | YYYY-MM-DD, 전부 채워짐 |
| `gu` | 자치구 | 25개 전부 |

| `price` | 매매가(만원) | **매매만** 채워짐 |
| `deposit` | 보증금(만원) | **전세·월세만** 채워짐 |

| `price_per_pyeong` | 매매 평당가 | 매매만. 우리는 안 씀 |

- **금액 단위는 만원**입니다. `56000` = 5억 6,000만원.
- 전세는 `deposit`만, 월세는 `deposit`+`monthly_rent`를 씁니다. 매매 데이터는 이 대시보드에서 쓰지 않습니다.

### 데이터 정제 (요약본 만들기 전에 처리)
| 처리 | 대상 | 처리 방법 |
|---|---|---|
| 완전 중복 줄 제거 | 16,882건 (모든 칸이 똑같은 줄) | 1건만 남기고 제거 |
| 앞뒤 안 맞는 줄 제거 | 월세인데 월세=0원 12건 | 제거 |
| 매매 제외 | 72,459건 | 이 대시보드에서 미사용 |

### 대시보드에 실제로 넣는 것 = 요약본
원본 32만 줄은 넣지 않습니다. 아래 단위로 **미리 계산한 요약 숫자만** HTML에 넣습니다.

- 쪼개는 기준: **자치구 × 법정동 × 면적대 × 계약월 × 거래종류(전세/월세)**
- 각 칸마다 저장하는 값: 거래 건수, 보증금 중앙값, 월세 중앙값(월세일 때), 보증금 25%값·75%값
- 면적대 구간(전용면적 기준, 제안):

  | 이름 | 범위 | 평수 감각 |
  |---|---|---|
  | 초소형 | 40㎡ 미만 | ~12평 |
  | 소형 | 40~60㎡ | 12~18평 |
  | **중소형** | 60~85㎡ | 18~26평 (신혼 주력) |
  | 중형 | 85~102㎡ | 26~31평 |
  | 대형 | 102㎡ 이상 | 31평~ |

  → 85㎡는 국민주택 기준선이라 경계로 잡았습니다. 구간을 바꾸고 싶으면 말씀해 주세요.

## 3. 핵심 화면과 기능

화면은 하나의 페이지 안에서 위→아래로 이어지는 4개 구역으로 구성합니다. (탭 전환이 아니라 스크롤)

### 화면 1 — 예산으로 시작하기 (맨 위)
- 신혼부부가 가장 먼저 하는 질문에 답하는 구역.
- 입력: **보증금 예산**(슬라이더 또는 입력칸), **거래종류**(전세/월세), **원하는 크기**(면적대)
- 출력: 그 조건으로 **갈 수 있는 자치구 목록**을 시세 낮은 순으로. "이 예산이면 15개 구까지 가능합니다" 식.
- 기본값: 전세 / 중소형(60~85㎡) / 예산 5억 6,000만원(전체 전세 중앙값)에서 시작.

### 화면 2 — 동네끼리 비교하기
- 자치구별(원하면 그 안의 동별) 전세·월세 대표 시세를 **가로 막대**로 나란히.
- 정렬: 시세 낮은 순 ↔ 높은 순 전환.
- 각 막대에 중앙값 + 거래 건수 표시. 건수가 적은 동네는 흐리게(아래 4번 지표 정의 참고).

### 화면 3 — 흐름 보기 (오르나 내리나)
- 선택한 지역·면적대·거래종류의 **12개월 시세 추이**를 꺾은선으로.
- "지난 1년간 이 동네 전세 중앙값이 ○○ → ○○로 ▲오름/▼내림" 한 줄 요약을 그래프 위에 표시.

### 화면 4 — 조건 좁혀 보기
- 구 · 동 · 면적대 · 거래종류를 골라 최종 후보를 좁히는 구역.
- 출력: 대표 시세(중앙값), 흔한 가격 범위(25%~75%), 거래 건수.
- 데이터가 부족한 조합이면 "이 조건은 거래가 적어 시세가 정확하지 않습니다"라고 안내.

### 모든 화면 공통
- 데이터가 없을 때: "조건에 맞는 거래가 없습니다" 안내 문구.
- 화면 어딘가에 "이 값은 지난 12개월 실거래 기록의 중앙값입니다. 실시간 매물이 아닙니다" 안내를 상시 노출.

## 4. 지표 정의

숫자를 어떻게 계산하고 어떻게 부를지 미리 못 박습니다. (계산 방식이 흔들리면 화면 숫자를 믿을 수 없게 됩니다.)

| 지표 | 정의 | 왜 이렇게 |
|---|---|---|
| **대표 시세** | 해당 조건 거래들의 **보증금 중앙값** | 평균은 초고가 몇 건(전세 최고 138억)에 끌려감. 중앙값이 "보통 이 정도"에 가까움 |
| **흔한 가격 범위** | 25%값 ~ 75%값 | 가운데 절반이 이 안에 있다는 뜻. 시세의 폭을 보여줌 |
| **거래 건수** | 그 조건에 해당하는 실거래 줄 수 (정제 후) | 시세를 얼마나 믿을 수 있는지 가늠자 |
| **최소 표본 기준** | 한 조합의 거래가 **5건 미만**이면 시세를 흐리게 표시하고 "표본 적음" 표시 | 5건 미만에서는 중앙값이 크게 흔들림 |
| **월세 대표값** | 월세는 보증금 중앙값 **+** 월세 중앙값을 함께 표기 | 보증금만으로는 월세 부담을 알 수 없음 |
| **흐름(추이)** | 계약월별 중앙값의 12개월 나열 | 개별 등락이 아니라 큰 방향을 봄 |

- 표기 규칙: 금액은 "5억 6,000만원"처럼 한국식. 면적은 "84㎡ / 25평"처럼 병기.

## 5. 기술 스펙

| 항목 | 내용 |
|---|---|
| 형태 | **단일 HTML 파일 1개** (`dashboard.html` 등, 이름은 만들 때 확정) |
| 실행 | 파일을 더블클릭 → 브라우저에서 바로 열림. 설치·서버 불필요 |
| 데이터 | 요약본을 HTML 안에 직접 포함 (인터넷 연결 없이 열림) |
| 범위 | 내 컴퓨터 로컬 전용. 인터넷 공개 안 함 |
| 기기 | PC 기본, 휴대폰 화면에서도 깨지지 않게 |
| 언어 | 화면 전체 한국어 |
| 외부 연결 | 없음 (지도 API·글꼴 CDN 등 바깥 서버를 부르지 않음) |

### 만드는 순서 (합의 후)
1. **요약본 만들기** — 원본 정제 → 요약 숫자 계산 → 확인. (산출물: 중간 요약 파일 또는 HTML에 바로 삽입)
2. **HTML 화면 만들기** — 위 4개 구역. (산출물: HTML 파일 1개)
3. 각 단계 후 브라우저에서 무엇을 확인하면 되는지 안내.
