# 스킬: 내 반복 작업을 클로드에게 자산으로 넘기기

> **스킬(Skill)** 은 반복되는 작업 절차를 한 번 정의해 두면, 필요할 때 클로드가 알아서 꺼내 쓰는 기능입니다.

데스크톱 앱에서는 Word·PDF·PowerPoint 같은 기본 탑재 스킬을 말 한마디로 부립니다.
한 걸음 더 나아가 **나만의 스킬**을 만들면, 내가 매번 똑같이 하던 셋업·정리·변환 절차를
슬래시 명령 하나로 묶을 수 있습니다. 이 영역은 주로 **Claude Code**(터미널 속 클로드)에서 다룹니다.

## 만들어 보는 예제

| 스킬 | 한 줄 | 가이드 |
|------|------|--------|
| `/new-project` | 새 프로젝트를 일관된 네이밍·git 셋업으로 시작 | [바로가기](./new-project/) |

_예제는 계속 추가됩니다._

## 스킬은 어디에 두나

Claude Code는 두 위치에서 스킬을 읽습니다.

- **개인 전역** `~/.claude/skills/<이름>/SKILL.md`: 모든 프로젝트에서 쓰임
- **프로젝트 한정** `<프로젝트>/.claude/skills/<이름>/SKILL.md`: 그 레포에서만 쓰임

`SKILL.md` 한 장이면 스킬 하나가 됩니다. 맨 위 YAML 프론트매터(`name`·`description`)와
그 아래 클로드에게 주는 지시문(절차)으로 구성됩니다. 자세한 해부는
[`new-project` 가이드](./new-project/)에서 실제 파일로 설명합니다.
